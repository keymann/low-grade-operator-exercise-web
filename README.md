# 일일수학 클론 — 연산 연습 웹앱

[일일수학(11math.com)](https://www.11math.com/calc) 스타일의 초등 연산 문제지를 클론한 웹앱입니다.
학생은 **숫자 패드(numberpad)로 답을 입력**해 채점하고, 부모님은 학년군·세부 항목별로 문제를 출제하거나 요일별 스케줄을 관리하고 학습 성취도를 확인할 수 있습니다.

Cloudflare Workers로 배포하며, 앱 상태는 **Cloudflare KV** 에 **회원(아이디)별** 문서로 저장됩니다. 같은 계정으로 로그인하면 여러 브라우저에서 동일한 상태가 공유됩니다. 모바일/태블릿/데스크탑 반응형입니다.

## 계정 / 로그인

- **최초 진입 시 로그인 화면**(아이디·비밀번호). 로그인 정보는 브라우저에 캐시되어 다음 방문 시 **자동 로그인** → 해당 아이디의 학생 모드로 진입합니다.
- **회원가입**: 아이디(한글·영어·숫자, 10자 이내) · 비밀번호 · 비밀번호 확인 · **초대 코드**(`hellow~!!!`)로 가입. 이미 있는 아이디·비밀번호 불일치·초대코드 불일치 시 안내 메시지를 보여줍니다.
- 모든 데이터(출제·스케줄·정답·제출기록·성취도)는 **회원별로 분리**됩니다.
- 기존 단일 상태는 아이디 **`시현엄마`** / 비밀번호 **`kw20021163`** 계정으로 자동 마이그레이션됩니다.
- 부모모드 진입 비밀번호는 로그인과 **별개**이며(계정별), 부모모드 **설정 탭에서 로그아웃**할 수 있습니다.

## 주요 기능

### 학생 모드 (로그인 후 기본 화면)
- **오늘의 문제** 타이틀만 노출 (다른 메뉴 없음)
- 답란을 누르면 **숫자 패드 팝업** — 헤더를 롱탭(길게 눌러) 드래그하면 화면 위에서 이동, 닫기(✕) 버튼 제공
  - 숫자/소수점/지우기 입력이 **즉시 답란에 반영**
- 모든 답란을 채우면 **제출 / 다시 풀기** 팝업
- **제출** 시 채점하고 틀린 문제를 빨간색으로 표시
- 모두 정답이면 축하 팝업

### 부모님 모드 (비밀번호 진입, 초기값 `kw20021163`)
- **출제하기**: 학년군 · 세부 항목 · 문제 개수를 선택해 즉시 출제 / **출제 문제 삭제**(확인 후 삭제)
- **스케줄**: 요일별 반복 출제 설정. 편집 시 *이 날짜만 / 반복 등록* 변경 + 항목·개수 조정
- **정답확인**: 출제한 문제의 정답지 및 학생 채점 결과 확인
- **제출기록**: 제출 시간 · 점수 · 틀린 문제 번호 이력
- **성취도**: 일/월 단위로 정답률·학습량 집계 (날짜·월 선택)
- **설정**: 비밀번호 변경

> 문제는 원본 페이지의 콘텐츠를 복사하지 않고, **같은 난이도 수준으로 매번 새로 생성**하며, 세트 내·과거 출제와의 **중복을 회피**합니다. 다자리 정수 덧셈·뺄셈·곱셈은 **세로식(세로셈)** 으로 출제됩니다.

## 커리큘럼 범위

현재 **3학년 1학기 ~ 6학년 2학기**(학기별 8개 그룹)가 구현되어 있습니다. 분류 체계는 `lesson.jpeg`(3~4학년군) · `high_lesson.jpeg`(5~6학년군) 일일수학 화면을 근거로 합니다. 1~2학년군은 추후 추가 예정입니다(`public/problems.js` 의 `GROUPS` 에 그룹/항목 추가). 시각·작도형 항목(예: 쌓기나무 모양)은 numberpad로 답할 수 없어 제외했습니다.

## 구조

```
public/
  index.html       # SPA 셸
  styles.css       # 반응형 스타일 (세로셈/numberpad/성취도 포함)
  problems.js      # 커리큘럼 카탈로그(GROUPS) + 문제 생성기
  numberpad.js     # 드래그 가능한 숫자 패드(답 입력)
  app.js           # 모드/상태/학생·부모 흐름 + KV 동기화
src/
  index.ts         # Worker: 정적 자산 서빙 + /api/state (KV GET/PUT)
  types.ts
```

상태(비밀번호, 스케줄, 출제/채점 결과, 제출기록)는 **Cloudflare KV** 에 저장되며(`/api/state`), localStorage 는 오프라인 캐시/폴백으로만 사용됩니다.

## 개발 / 실행

```bash
npm install
npm run dev      # wrangler dev (로컬 KV 시뮬레이션 자동 사용)
npm run check    # 타입체크 + 배포 dry-run
npm run deploy
```

> **배포 전** KV 네임스페이스를 만들고 `wrangler.jsonc` 의 `kv_namespaces[].id` 를 실제 값으로 교체하세요:
> ```bash
> wrangler kv namespace create STATE
> ```
> 로컬 `wrangler dev` 는 Miniflare 로컬 KV 를 사용하므로 placeholder id 로도 동작합니다.
